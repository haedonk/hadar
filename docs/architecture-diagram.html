Hadar Architecture

Hadar collects smart home Zigbee sensor data through MQTT, stores structured readings in PostgreSQL, and runs a scheduled anomaly detection workflow in k3s.

Data Flow

  • Zigbee devices publish energy and temperature data through Zigbee2MQTT.
  • The ingestion worker subscribes to MQTT topics and maps payloads into structured events.
  • Repository and service layers write normalized readings and metadata into PostgreSQL.

Deployment

  • k3s runs the project in the hadar namespace.
  • The ingestion pipeline runs as a Deployment.
  • The isolation forest pipeline runs as a scheduled CronJob.
  • Secrets are created outside Git and referenced by workload manifests.

Storage and Analysis

  • PostgreSQL stores device, reading, load group, and error log tables.
  • A retained hostPath PV backs shared runtime storage through hadar-data-pvc.
  • The anomaly detection job reads historical data and produces model/output artifacts locally.
